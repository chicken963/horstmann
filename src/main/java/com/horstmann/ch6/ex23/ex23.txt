Во врезке "Внимание!" из раздела 6.6.7 упоминается вспомогательный метод throwAs(), применяемый для "приведения" типа
исключения ех к типу RuntimeException и его генерирования.
Почему для этой цели нельзя воспользоваться обычным приведением типов, т.е. throw(RuntimeException) ex?

Потому что RuntimeException не является супертипом проверяемых исключений
